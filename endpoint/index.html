开发人员依靠屏蔽实现视觉一致性和强大功能 自定义选项。作为服务提供者或数据提供者，您可以 使用端点标记提供内容，同时向用户提供完整的 盾徽定制的力量。

使用端点标记，可以通过 JSON端点。内容可以预先呈现，也可以动态生成。 在响应和带宽利用之间取得平衡 一方面，另一方面，缓存行为是可配置的， 以防护罩最小值为准。端点URL提供给屏蔽 通过查询字符串。徽章精灵ELVNS获取并格式化徽章。

端点徽章比重定向到 静态徽章端点或在服务器上生成SVG：

  1. 内容和演示是分开的。 服务提供者编写徽章，徽章精灵 ELVNS 从要格式化的用户。 作为一个服务提供商，你编写了徽章，但是 不用担心自己的造型。你甚至不必 将格式化选项传递给屏蔽。
  2. 徽章格式始终是100%最新的。不需要跟踪 NPM包、徽章模板或选项的更新。
  3. JSON响应易于实现；比HTTP重定向更容易实现。它 在几乎任何框架中都是微不足道的，并且更兼容 托管环境 RunKit endpoints.
  4. 作为服务提供商，您可以依赖徽章精灵的 CDN。没必要 研究HTTP头。调整缓存行为非常简单 在JSON响应中设置属性。

Schema

中断对架构的更改将触发 `架构版本`。

schemaVersion 架构版本
Required. Always the number 1.
label 标签类
Required.左边的文本或空字符串，以省略 徽章.这可以被查询字符串重写.
message 自定义内容
Required. 不能为空.正确的文本格式.
color 颜色
默认: 浅灰色. 正确的颜色。支持8个 上面命名的颜色，以及命名的十六进制、RGB、RGB、HSL、HSLA和CSS 颜色。这可以被查询字符串重写
labelColor 标签颜色
默认: 灰色. 左边的颜色。这可以被覆盖 查询字符串。
isError
默认: false. 如果true 将此视为错误 徽章。这样可以防止用户覆盖颜色。未来 它可能会影响缓存行为。
namedLogo 名称图标
默认: 无. 徽章精灵支持的命名徽标之一 or simple-icons. 可以 通过query string直接使用
logoSvg
默认: 无. 包含自定义徽标的SVG字符串.
logoColor
默认: 无.与查询字符串的含义相同。可以使用query string重写.
logoWidth 图标宽度
默认: none. 与查询字符串的含义相同。可以被重写的query string.
logoPosition 标志定位
默认: none. 与查询字符串的含义相同。可以被重写的query string.
style
Default: flat. 与查询字符串的含义相同。可以被重写的query string.
cacheSeconds 缓存秒数
Default: 300, min 300. 设置HTTP缓存 以秒为单位的寿命，应该受到Shields的cdn和 下游用户。低于300的值将被忽略。这可以让你调整 性能和流量与响应。您指定的值可以是 由用户通过查询字符串重写，但只覆盖较长的值.
